  <action>
    1. `Dockerfile`: Python 3.12-slim base.
       - Stage 1 (builder): COPY requirements.txt, pip install --no-cache-dir -r requirements.txt
       - Stage 2 (runner): Copy installed packages from builder, COPY app/ into /app/app/, set WORKDIR /app.
       - CMD: `["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "3001"]`
       - EXPOSE 3001
       - HEALTHCHECK: `CMD curl -f http://localhost:3001/health || exit 1` (interval 30s, timeout 10s, retries 3, start-period 40s)
       - Install curl in the image for healthcheck (apt-get install -y --no-install-recommends curl && rm -rf /var/lib/apt/lists/*).

    2. `docker-compose.yml`: Follow exact Phase 1 Traefik pattern.
       ```yaml
       services:
         whatsapp-bot:
           build: .
           restart: unless-stopped
           networks:
             - n8n_default
           expose:
             - "3001"
           env_file:
             - .env
           labels:
             - "traefik.enable=true"
             - "traefik.http.routers.whatsapp-bot.rule=Host(`bot.srv1175749.hstgr.cloud`)"
             - "traefik.http.routers.whatsapp-bot.tls=true"
             - "traefik.http.routers.whatsapp-bot.tls.certresolver=myresolver"
             - "traefik.http.services.whatsapp-bot.loadbalancer.server.port=3001"
           healthcheck:
             test: ["CMD", "curl", "-f", "http://localhost:3001/health"]
             interval: 30s
             timeout: 10s
             retries: 3
             start_period: 40s
       networks:
         n8n_default:
           external: true
       ```
       No host port binding -- Traefik discovers via Docker socket.

    3. `.dockerignore`: Exclude tests/, __pycache__/, .env, .git/, *.md, .pytest_cache/, .mypy_cache/, *.pyc.

    4. Test Docker build locally: `docker build -t whatsapp-bot-test .` (should succeed without errors).

    5. Push whatsapp-bot/ to the telegram-bot-agency GitHub repo.

    6. Deploy to VPS via SSH:
       - SSH into VPS
       - `cd /opt && git clone <repo> telegram-bot-agency` (or git pull if already exists)
       - `cd /opt/telegram-bot-agency/whatsapp-bot`
       - Create `.env` file with all required env vars (TWILIO_AUTH_TOKEN, TWILIO_ACCOUNT_SID, GEMINI_API_KEY, GOOGLE_CREDENTIALS_JSON, AMADEUS_CLIENT_ID, AMADEUS_CLIENT_SECRET, WEBHOOK_BASE_URL=https://bot.srv1175749.hstgr.cloud)
       - `docker compose up -d --build`
       - Verify: `curl -s https://bot.srv1175749.hstgr.cloud/health` returns 200

    Note: If env vars are not yet available (Twilio credentials pending per STATE.md blockers), the container should still start and /health should return 200. Webhook calls will fail gracefully until credentials are configured.
