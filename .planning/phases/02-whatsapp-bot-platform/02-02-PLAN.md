    1. `app/sheets/client.py`: ProductSheetsClient class.
       - Constructor takes spreadsheet_id and optional credentials_json (from env). Uses `Credentials.from_service_account_info(json.loads(credentials_json))` with readonly scope. Builds sheets v4 service with `cache_discovery=False`.
       - `search_product(query: str) -> list[dict]`: Reads from configured range (default Sheet1!A:D). Headers from first row. Case-insensitive substring match on "nombre" column. Returns list of dicts with all columns.
       - `_format_price(price_str: str) -> str`: Format as "$X,XXX MXN" with comma separators and MXN suffix per user decision.
       - Cache product data in memory with 10-minute TTL (same pattern as existing GoogleSheetsManager). Use `_cache` dict with `_cache_time` float.
       - Wrap sync Sheets API calls in `asyncio.get_event_loop().run_in_executor()` for async compatibility per research pitfall #4.

    2. `app/handlers/product.py`: ProductHandler extends BaseHandler.
       - `can_handle()`: Use keyword detection for product-related Spanish phrases: "precio", "cuesta", "tienen", "disponible", "producto", "busco", "quiero", "comprar", etc. Return True if any keyword matches. This is a first-pass filter -- Gemini handles nuance in the response.
       - `handle()`: Search products via ProductSheetsClient. If matches found, inject product data into the Gemini prompt: send message like "El cliente pregunta: '{message}'. Productos encontrados: {formatted_products}. Responde mencionando los productos con sus precios y disponibilidad de forma conversacional. Pregunta si quiere hacer un pedido." Then return Gemini's response. If no match, send "El cliente pregunta por '{message}' pero no encontre productos similares. Sugiere que contacte directamente o pregunta si busca algo diferente."
       - Constructor takes ProductSheetsClient instance.

    3. `app/handlers/order.py`: OrderHandler extends BaseHandler.
       - `can_handle()`: Returns True when session.state has `order_flow=True` (set by product handler when customer says yes to ordering), OR when message contains ordering intent keywords ("pedido", "ordenar", "comprar", "quiero pedir").
       - `handle()`: Multi-step state machine in session.state:
         - Step 1 (no state): Ask for name -> set state order_step="name"
         - Step 2 (name): Store name, ask for product -> set order_step="product"
         - Step 3 (product): Store product, ask for quantity -> set order_step="quantity"
         - Step 4 (quantity): Store quantity, write to "Pedidos" sheet tab (append row with timestamp, name, product, quantity, phone, status="Nuevo"), confirm to customer, clear order state.
       - For MVP, write to Sheets "Pedidos" tab. If Sheets write fails, still confirm to customer and log the error (don't lose the order info in the response).

    4. `app/dispatcher.py`: Create `handler_registry` dict mapping handler name strings to handler classes/factories: {"product": ProductHandler, "order": OrderHandler, "flight": FlightHandler (placeholder import), "qa": QAHandler}.
       - `async def dispatch(business: BusinessConfig, session: ConversationSession, message: str, sheets_clients: dict, amadeus_provider) -> str`: Iterate business.handlers list, instantiate/get handler, call can_handle(), if True call handle() and return. If none match, fall through to QAHandler.
       - Handlers are instantiated with their dependencies (ProductHandler gets sheets_client, FlightHandler gets amadeus_provider, etc.).

    5. Update `app/handlers/__init__.py` to export all handlers.

    6. Write tests FIRST. Mock Google Sheets API responses for ProductSheetsClient tests. Mock Gemini chat for handler tests. Test dispatcher routing logic with mock handlers.
